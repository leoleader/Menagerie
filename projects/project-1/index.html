12/10 - Argos Dev Update

A few months later and the MVP for Argos is completed! Our team (which at this point is pretty much just Peyton, Wasif, and myself) has a few adjustments and tweaks here and there left but overall the MVP is pretty much done, which is super exciting.

Personally I have contributed 1000+ lines of code and developed:

- Rest API Calls and CRUD operations
- Routes and Controllers for API Calls
- System to continuously parse received server messages from car 
  and populate the database with the data
- Historical and More Data buttons
- Battery

Pictured above is a look at the data graph for a sample run, currently displaying data for the Pack Temperature. The graph smoothly demonstrates the change in the pack temp throughout the run and allows users to pinpoint values at exact times. In the sidebar the user can select one of the data nodes in the database and a datatype and then display that data in the graph, allowing users to access any and all available data for a run.

Pictured here is a look at our mobile version of Argos, specifically taking a look at the functionality of the Historical and More Details Button on the real time data graph page. When the Historical Button is pressed a carousel displaying the Run History appears, which allows the user to click through and select a Run from the database to look at.

Above is a peek at my most recent personal addition to Argos, a battery component that displays the current charge of the battery. Accidentally leaving the battery on overnight, forgetting to charge it causing it to die during the middle of the day, these kind of situations are a nuisance and by having an easy to check display of the battery charge at all times, it will help to avoid these things from happening in the future.

09/01 - Development Begins

This semester I am excited to announce that I will be working as a Software Engineer for the Data Visualization and Analysis Team for Notheastern Electric Racing! We are going to be working on a few different really cool projects to help our engineers access and understand data from the car so that they can better improve it.

Currently in development is a project called Argos, essentially a telemetry hub that lets users query any and all data from the car’s server, then organizes and presents it in a user-friendly series of charts and figures.

Argos uses Express backend in TypeScript, along with Prisma for easier/more efficient querey building and a SQLite database. Server communication is being done using socket IO, and the front end is using Angular. We initially had the project’s frontend in React but early on we discussed the pros/cons of React vs. Angular for this project and decided to go with Angular. Finally we are using ApexCharts for our charting library.

As the project is continuously developed I will update this article with progress reports :)
