Routers, Protocols, Crawlers Oh My!

April 1, 2023

In this post we will be looking at two of my favorite projects from a class that I have really been enjoying recently, Networks and Distributed Systems!

The first project we will be looking at is a simplified virtual BGP router. The professors provided us with a simulator and several configs, in which the router we built must be ran against and pass in correctness and performance. Responsibilities of the router included accepting route announcements from simulated peer routers, generating new route announcements for peer routers, managing and compressing forwarding tables, calculating optimal routes, and forwarding data packets that arrive from simulated internet users. A big part of the project was properly aggregating the forwarding table entries as to optimize performance.

Now this project would be much easier with the usage of libraries and modules that deal with ip addresses like Python’s ipaddress module, however the usage of libraries was severely limited, meaning that we had to program methods to manipulate ip addresses from scratch. This project gave me a lot of hands on experience working with and significantly strengthened by understanding of ip addresses, subnets, network masks, CIDR notation, and the core infrastructure of the internet as a whole.

The program initializes, connects to sockets, sends handshakes, receives and forwards route announcements, sends data, and then dumps it’s routing table. Pictured above is the verification of the data message counts at each node and verification that the routing table is correct and properly aggregated.

Currently the code for this project and the next are in private repositories on my github that I plan on making public once the class is finished.

The second project that I am highlighting is a web crawler for a fake social media site called Fakebook that the professors created. Fakebook is a barebones site with tens of thousands of user profiles each with varying amounts of friends and different personal information on their profiles. Our goal was to explore user profiles, downloading HTML pages and parsing them until we found five student-specific secet flags.

Obviously there is no way to know what five pages of the thousands on the site contain these flags, so the crawler would need to crawl potentially all of these pages, meaning that it would have to do so efficiently and quickly. In addition, the crawler needed to implement HTTP/1.1, handle various HTTP status codes, and avoid getting stuck in cycles by keeping track of profiles that had already been explored.

I started by going into dev tools and studying the GET and POST HTTP requests that were sent as I logged in and explored Fakebook, as pictured below:

Once I familiarized myself with the headers and structure of HTTP requests and how the site functioned, I started programming. The cralwer first needed to connect with a TCP socket wrapped in TLS, and log in to the site using server, port, username, and password parameters provided in the command line. In order to login, the crawler needed to send GET and POST requests to obtain the necessary csrftokens and session ids, and once it succesfully logs in, is free to start exploring the website. I mainly used regex for parsing along with good ol split and used sets for my data structures to easily make sure the crawler never visited the same profile twice.

Getting the crawler to work was one thing, but optimizing was a whole nother. When I first succesfully ran my crawler, it took nearly 30 minutes, which was just barely fast enough not to timeout during the submission, however was definitely not ideal. To improve the performance I integrated connection keep-alive, chunked encoding, gzip encoding (compressing and decompressing the HTML content), and spruced up the algorithm the crawler used to explore profiles with. This was the more challenging half of the project, though my efforts were most certainly not for nothing because the performance time dropped from 30 minutes to a little bit over 1 minute 30 seconds. This project really opened my eyes to how useful web crawlers can be and I am excited to use this experiecne to build my own crawlers in the near future.
